### 2026-07-14 (Phase 1) — Checkpoint 1.7: data-quality gate (FR-6)
**Shipped.** `IDataQualityGate`/`DataQualityGate` — a **pure** evaluator (no DB reads/writes) over a security's candidate bars returning a `QualityReport` of `QualityFlag`s (issue + `Warn`/`Reject` severity + date + detail). Three FR-6 checks: **gaps** (an expected trading session with no bar — the expected-session set is a caller input; the `ICalendarService` that *produces* it is FR-30/checkpoint 1.8); **field integrity** — a null/NaN/inf or non-positive **close**, a bad open/high/low, **or a present-but-corrupt adjusted close** → `Reject` (fail closed, rule 10 — see the review note); **robust return outliers** — a daily (adjusted) return whose **median/MAD robust z** exceeds `Data.OutlierZ` (=8.0), the scale **floored at 0.1%/day**; and **adj/raw reconciliation** — the `adj_close/close` factor is piecewise-constant absent a corporate action, so a factor step in `(prev.date, cur.date]` with no dividend/split in the feed raises the `UnexplainedAdjustment` **alarm** (a missing corporate action). `DataQualityOptions` (SectionName `Data`, keys `OutlierZ`/`BarCrossCheckSampleSize`/`BarCrossCheckTolerancePct` matching CONFIG). The **Alpaca bar cross-check is a DORMANT seam** — `IBarCrossCheckProvider` + `AlpacaBarCrossCheck` that **fails loud** (no Alpaca account at launch; the Secrets Alpaca pair is optional, NFR-4; the two `BarCrossCheck*` keys are wired-but-inert). Registered `IDataQualityGate` in `AddAlphaLabMembership`. **No migration, no invented schema column or config key** (rule 14) — the reconciliation epsilon (0.05%) and the robust-scale floor (0.1%/day) are documented internal constants + stop-and-report seams.
**Two implementation calls (real-data honesty, not silently fixed).** (1) The outlier test needs a **robust** median/MAD z, not a plain sample z — a lone 12σ print self-masks (it inflates the std and scores ~4σ on a plain z), and the scale is **floored** so a near-constant/illiquid series (MAD≈0) can neither manufacture outliers from ordinary moves *nor* mask a real 40% bad print (a lone spike among flat days also has MAD≈0). A test demonstrates the plain-z self-masking directly. (2) Reconciliation is **presence-based** (a step is explained by the *presence* of any dividend/split in the window, not by matching the step magnitude) — deliberately conservative (zero false positives; only alarms when there is NO event) because the magnitude-aware residual check needs the exact `adj_close` factor arithmetic, which has no byte-real fixture to validate against yet (building it now would risk false positives on real data).
**Verified.** Data tests **112** (was 93); `tools/ci.ps1` green (**134** total; no migration → `SchemaFidelityTests` still 14 tables; bars-append-only + full reference-graph + no-secrets guards clean). **FX-QualityGate:** one synthetic series with a gap day, a NaN close, a +37% outlier print, a dividend-explained factor step (clean), and an unexplained factor step → **exactly four flags** (MissingBar/NanField/OutlierReturn/UnexplainedAdjustment), the dividend step not flagged, `HasRejects` driven only by the NaN reject, and the **same flag set on shuffled input** (order-independence).
