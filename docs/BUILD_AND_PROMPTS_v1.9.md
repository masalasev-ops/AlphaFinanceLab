*Companion to MASTER_DESIGN_v1.9. - **Phase 6 adds the French factor/RF ingestion (D41), Ledoit–Wolf covariance (D42), the parameterized cost model was already live from Phase 2 (D43) — Phase 6 adds its capacity readout — and the trade-level track (D44).**

## 1. Functional requirements (numbered; tests cite these)

**Data layer**
- FR-1 Ingest daily raw+adjusted bars for the configured universe from **EODHD** behind `IMarketDataProvider`; backfill-then-delta; completed bars only.
- FR-2 **Versioned append-only bars (D40):** corrections insert new `(security_id, date, version, observed_at)` rows; no UPDATE/DELETE ever (CI grep); every run records its **watermark**; all reads resolve latest-version ≤ watermark.
- FR-3 **Security master (D39):** permanent `security_id` keys everywhere; `ticker_history` aliases; corporate-action feed ingested and typed (dividend, split, ticker change, merger cash/stock/mixed, spin-off, delist).
- FR-4 **Index membership (D35/D49/D70):** daily membership refresh cross-checked between two named sources, fail closed on divergence (agreement ⇒ flip `in_index`+dates; divergence ⇒ hold state + alert). **Launch wiring per D49:** IVV holdings CSV **primary** + Wikipedia scrape **cross-check**; the EODHD constituents provider is built but stays **dormant** behind `Universe.MembershipPrimary` (the endpoint is not on the launch tier). Count sanity 495–510 (S&P 500); the D70 S&P 100 slice runs its own pair (OEF holdings CSV primary + Wikipedia S&P 100 cross-check, count sanity 99–103). **Historical constituents for as-of reconstruction ingest the fja05680/sp500 community CSV at launch (D49/D70; EODHD historical snapshots post-upgrade).**
- FR-5 **Reference data:** EODHD sector/industry per security (change-logged); **Ken French daily factors + RF** monthly refresh with checksum/continuity validation (D41).
- FR-6 Data-quality gate: gaps/NaNs/outliers; rotating-sample bar cross-check vs Alpaca; dividend/split reconciliation vs adjusted/raw ratios.
- FR-7 **Multi-day catch-up (D47):** ordered per-day replay (bars → corp actions → membership as-of → funnel), one ACID transaction per day, resumable, idempotent, logged to `catchup_log`.

**Funnel & ledger**
- FR-8 The six-stage funnel per MASTER_DESIGN_v1.9 §6; Stage 1 shared; Stage 3 zero-score invariant; Stage 4 closes only via `ExitPolicy` + forced events; decide-at-close/fill-next-open.
- FR-9 Ledger conventions (D30) + full corporate-action semantics (§13.6): raw-price trading, ex-date dividends, split adjustments, ticker-change continuity, merger conversion/cash-out, spin-off receipt with basis allocation, delist force-exit, unmapped-event freeze+alert.
- FR-10 **Cost model (D43):** commission + half-spread bucket + √impact, participation cap 2% ADV with rejection logging; coefficients config; cost-model version stamped per trade; costs always on.
- FR-11 Sizing: inverse-vol under a portfolio vol target using **Ledoit–Wolf covariance (D42)**; guardrails per DESIGN_IMPROVEMENTS_v1.9 §3.4, fail closed.

**Evaluation & control**
- FR-12 **Random populations (D36):** M=200 seeded members per cadence family + M=50 cost-free; lightweight ledger-only accounts; deterministic under (familySeed, memberIndex, date); band + percentile services. **v1.9.7 (finding 115) — cost-match verification:** each evaluation persists realized annualized turnover for the strategy and its matched population (median + band); a strategy outside `Populations.TurnoverMatchTolerancePct` (default ±30%) of the population median renders a **cost-match caveat** on the S3 panel and in the `StrategyRow` read-model, and is logged; `matched_params_json` records a realized-turnover target so re-matching is possible. Rationale: the controls match exit *shape*, not signal autocorrelation — an unverified churn mismatch biases the percentile anti-conservatively (the strategy pays less cost drag than its own null), the one direction the lab must never err.
- FR-13 Metrics per DESIGN_IMPROVEMENTS_v1.9 §1: Jensen's alpha (NW errors, French RF), IR, expectancy, deflated Sharpe (trials registry), drawdown, turnover, capacity pressure.
- FR-14 **NW-corrected MDE (D48)** computed per comparison, persisted to `power_reports`, rendered beside every gap; the gate returns `TooEarly` inside it.
- FR-15 **Trade-level track (D44):** per-trade expectancy + moving-block bootstrap CI + trade-track MDE for flagged families; feeds S8.
- FR-16 Promotion gate: paired β-adj-alpha test on the evaluation cadence; monitor Suspect veto; verdicts Promoted/Refused/TooEarly; revert path; go-live log.
- FR-17 Ensemble allocator: banded, slow, logged; Suspect freeze; TooEarly tilt cap.
- FR-18 Overfitting Monitor: eight signals per OVERFITTING_MONITOR_v1.9, incl. **S3 population percentile**; statuses wired to gate/allocator; auto-retire.
- FR-19 **Arena Replay (D37):** full pipeline on a simulated clock, as-of membership, `run_kind='replay'`, absolute quarantine (query-layer + test), machinery-validation suite, threshold-calibration report; seeding mode = walk-forward engine. **v1.9.7 (findings 113–114):** the validation suite additionally records the **edge-plant survival** fraction (D64 edge plants still promotable at 5y/10y; Phase-4 DoD floor `Replay.EdgePlantSurvivalFloor5y`, default 0.90; every edge-plant auto-retire logged with its triggering signal — on a floor failure S6's *patience* is recalibrated, never the plant) and the **joint any-signal false-alarm** fraction for no-edge plants (bound `Replay.JointFalseAlarmMaxFrac`; per-signal contribution as a permanent report section).
- FR-20 Trials registry: every fork/sibling/retrain counted; replay trials in a separate column.

**LLM**
- FR-21 `IAnalysisProvider` on the **Message Batches API** for scheduled reads; prompt caching; per-task model tiering (config); `analysis_cache` keyed (prompt_hash, model, date); forward-only (absent in replay).
- FR-22 **News budget (D46):** relevance filter → dedupe → 25-article cap → 2,000-char truncation, enforced in `INewsProvider` before any token; D24 hard budget + degradation order unchanged.
- FR-23 Research-assistant endpoints (briefs, hypotheses, skeptic) as on-demand GUI actions — **resolved by D82 (v1.9.21; authoritative spec MASTER §23.4):** the hypotheses endpoint is KEPT (`POST /api/v1/analysis/hypotheses`); its output is a **draft `journal_entries` hypothesis** that must cite parent evidence (422 without); **only the operator pre-registers** (rule 30). The LLM's arena pricing runs through the contestant seat + its mandatory no-LLM twin (D81, Phase 6); the earlier blend-A/B framing is retired (finding 178).

**GUI & ops**
- FR-24 Blazor screens per MASTER_DESIGN_v1.9 §15, styled per `UX_DESIGN_SYSTEM_v1.9.md` (component anatomy) and `UX_GUIDELINES_v1.9.md` (tokens + UX-1…UX-14), incl. population bands, MDE lines, percentile chips, separation-state chips (D63), regime episode badges (D45), Data-health screen (cross-check status, watermark, catch-up log, factor freshness), frozen-position flags. **Deferrable workstream per D65:** screens may lag their phases (API-only operation via Scalar is sanctioned until Phase 4 sign-off) with a hard deadline of Phase 7 exit; the underlying read-models and API endpoints are never deferrable.
- FR-25 Ops: ACID daily run; WAL; nightly backup; snapshot-before-migration; leakage suite + no-delete/no-update greps in CI; deterministic replays of any historical run at its watermark.

**Non-functional**
- NFR-1 Determinism = f(inputs, watermark, seeds).
- NFR-2 Every judged number reconstructible from persisted rows (auditability).
- NFR-3 Empty-database renders cleanly on every screen.
- NFR-4 Secrets only from the gitignored `appsettings.Secrets.json` (keys `Secrets:EodhdApiToken`, `Secrets:AnthropicApiKey`, optional Alpaca pair); config builder adds `appsettings.json` + `appsettings.Secrets.json` only — no env vars, no User Secrets (D67).

Guard: process only through the last *completed* session (`ProcessThroughLastCompletedSessionOnly`), never a half-run of today. Tests: two writers never overlap (a command during a simulated run is queued/409, not raced); the Worker runs the daily job with the API stopped (lab runs without any UI/API); **OnDemand launch after an N-day gap replays exactly the missed completed sessions in order and exits; a second launch the same evening is a no-op; a launch before today's close does not run today (D61).** **v1.9.7 (D72, findings 111–112):** the OnDemand launch order is **schema → catch-up → drain queued `jobs` (replay, analysis) → backup → exit** (`Worker.DrainQueuedJobsOnLaunch`); jobs never run inside a daily write transaction and catch-up always precedes them. The running Worker writes `worker_state.heartbeat_at` every `Worker.HeartbeatSeconds`; on launch it clears a stale `run_in_progress=1` (heartbeat older than `Worker.StaleRunThresholdSeconds`), marks the orphaned `runs` row `failed`, and logs the recovery; the Api treats a stale heartbeat as not-in-progress for its 409 decision. Added tests: `FX-JobDrain` (a job enqueued via the Api executes on the next OnDemand launch, after catch-up); `FX-CrashedRun` (a stale `run_in_progress=1` is cleared on launch and no longer 409s the command path).

```markdown
# CLAUDE.md — AlphaLab (paper trading research lab)

## What this is
Personal C#/.NET paper-trading research lab. SQLite, EODHD data, Claude as a batched
text-reading assistant. Research only — never investment advice. The UI is a swappable
client of AlphaLab.Api (D57); honesty logic lives in read-models (D58), not the UI.

## Hard rules (violations are bugs, not style)
1. Forward paper P&L judges strategies. Replay (`run_kind='replay'`) judges only the
   machinery and is quarantined from every forward view, gate, and chart.
2. All identity is `security_id`. Tickers are aliases via `ticker_history`.
3. Bars are versioned append-only. Never UPDATE or DELETE a bar row (CI greps).
   Every run records a watermark; reads resolve latest-version <= watermark.
4. Point-in-time everywhere via IFeatureView(asOf, watermark). New features/labels
   get a leakage test in the same PR.
5. Costs always on (commission + spread bucket + sqrt-impact, participation cap).
   Cost-model version stamps every trade.
6. Alpha = beta-adjusted (Jensen's) vs the cap-weight account, NW errors, French RF.
   Never display a head-to-head gap without its NW-corrected MDE.
7. Strategies declare Horizon + ExitPolicy. Stage 4 closes only via ExitPolicy or
   forced corporate-action/guardrail events. Zero-score names are never selectable.
8. Frozen parameters: changing a live strategy forks a new strategy_id + trial row.
9. Random populations are the null: every strategy is ranked against its matched
   population. Cost-free population is display-only.
10. Fail closed: missing input -> reject order; unmapped corporate action -> freeze
    position + alert. Never silently mispriced.
11. Secrets from the gitignored `appsettings.Secrets.json` only — no env vars (D67). Never commit or log keys.
12. Each daily run = one ACID transaction; catch-up replays missed days in order.
13. Claude is forward-only, batched, budgeted; INewsProvider enforces the news
    budget before any token is spent.

## Workflow
- Plan Mode for anything multi-file; wait for plan approval.
- Strict milestone scope: implement only the current phase's FRs.
- Every phase ends with: tests green, demo, PROGRESS.md entry.
- Cite FR numbers in test names (e.g. FR10_ParticipationCap_RejectsAndLogs).

## Layout
src/AlphaLab.Core (domain + read-model DTOs) · src/AlphaLab.Data (EF, providers) · src/AlphaLab.Strategies ·
src/AlphaLab.Evaluation (metrics, gate, monitor, populations, replay, read-model builders) ·
src/AlphaLab.Llm · src/AlphaLab.Api (ASP.NET Core minimal-API — the UI boundary, D57) ·
src/AlphaLab.Web (Blazor client of AlphaLab.Api) · tests/ (mirrors src) · docs/
```
